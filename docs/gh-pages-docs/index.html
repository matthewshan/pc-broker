Introduction

LLM PC Broker is an internal service that sits between your homelab Kubernetes cluster and a gaming PC running Ollama. It handles power management (Wake-on-LAN), readiness polling, and request proxying — so internal applications always have a stable API endpoint regardless of whether the gaming PC is on or off.

A lightweight built-in UI provides a status dashboard, power controls, and a chat interface.

💡 This service is internal-only. It should never be exposed to the public internet. Access is via an internal gateway or Twingate.

        ▼
  llm-pc-broker  (Kubernetes pod)
  ├─ Background poll loop  ─────────────────────────────────────┐
  ├─ State machine (offline → waking → host_up → ready)        │
  ├─ Event ring buffer (last 200 events)                        │
  │                                                              │

  ├─ GET  /api/events       ← recent operational events         │
  └─ GET  /                 ← built-in chat + status UI         │
        │                                                        │
        ▼  (LAN)           polling every 5s ◄───────────────────┘
  Gaming PC
  └─ Ollama :11434

Wake-and-Wait Flow

When an LLM request arrives and Ollama is not ready:

LLM request arrives
  └─ Ollama reachable? ──yes──► proxy immediately
       └─ no
           └─ PC reachable? ──no──► send Wake-on-LAN (UDP broadcast)
               └─ poll PC reachability  (timeout: HOST_REACHABILITY_TIMEOUT)
                   └─ poll Ollama readiness (timeout: OLLAMA_READINESS_TIMEOUT)
                       ├─ ready ──► proxy request
                       └─ timeout ──► 503 + {"error": "warming_up"}

Operational States

The broker uses a unified state machine. The same state value appears in API responses, the UI dashboard, and logs.

offline → waking → host_up → ollama_starting → ready
any state → timeout / error
StateMeaning
offlinePC and Ollama are not reachable
wakingWoL packet sent; waiting for PC to respond
host_upPC is pingable; Ollama not yet ready
ollama_startingOllama port open; models may still be loading
readyOllama is healthy and models are available
timeoutWake or readiness attempt exceeded timeout
errorUnexpected error during polling

Quick Start

Run locally

pip install -r requirements.txt

PC_MAC=aa:bb:cc:dd:ee:ff \
PC_HOST=192.168.1.100 \
OLLAMA_BASE_URL=http://192.168.1.100:11434 \
uvicorn app.main:app --reload

Run with Docker

docker run -p 8000:8000 \
  -e PC_MAC=aa:bb:cc:dd:ee:ff \
  -e PC_HOST=192.168.1.100 \
  -e OLLAMA_BASE_URL=http://192.168.1.100:11434 \
  ghcr.io/matthewshan/llm-pc-broker:latest

Run tests

pip install -r requirements-dev.txt
pytest tests/ -v

Status API

GET /api/status Combined PC + Ollama status ▶

Returns the current state of the broker, PC reachability, and Ollama readiness.

Response

{
  "state": "ready",
  "pc": {
    "reachable": true,
    "last_seen": "2026-05-17T12:00:00Z"
  },
  "ollama": {
    "reachable": true,
    "models": ["llama3", "deepseek-r1"]
  }
}

Try it

Power API

POST /api/power/on Send Wake-on-LAN packet ▶

Sends a UDP broadcast WoL magic packet to the gaming PC. Idempotent — safe to call even if the PC is already on.

Returns 202 Accepted on success, 503 if PC_MAC is not configured.

Response (202)

{ "accepted": true, "message": "Wake-on-LAN packet sent" }

Try it

POST /api/power/off Request graceful shutdown ▶

Calls the optional shutdown agent running on the gaming PC. Returns 202 on success, 503 if the agent is not configured or unreachable.

⚠️ Requires SHUTDOWN_AGENT_URL to be configured.

Try it

LLM API

GET /api/llm/health Ollama readiness check ▶

Returns {"status": "ok"} when Ollama is reachable, 503 otherwise.

Try it

GET /api/llm/models List available Ollama models ▶

Proxies GET /api/tags from Ollama and returns model names.

Response

{ "models": ["llama3", "deepseek-r1"] }

Try it

POST /api/llm/chat Chat proxy with wake-and-wait ▶

Accepts an Ollama-format chat request. If the PC is offline, automatically wakes it and waits before proxying. Returns 503 with {"error":"warming_up"} if readiness times out.

Request body

{
  "model": "llama3",
  "messages": [
    { "role": "user", "content": "Hello!" }
  ],
  "stream": false
}

Try it

POST /api/llm/generate Raw generate proxy ▶

Proxies a raw generate request to Ollama. Same wake-and-wait behavior as /api/llm/chat.

Request body

{ "model": "llama3", "prompt": "Why is the sky blue?", "stream": false }
POST /api/llm/embeddings Embeddings proxy ▶

Proxies an embeddings request to Ollama.

Request body

{ "model": "llama3", "prompt": "The sky is blue." }

Events API

GET /api/events Recent operational events ▶

Kubernetes Deployment

Manifests live in the k3s-homelab repo under applications/llm-pc-broker/. They are picked up automatically by the ArgoCD applications ApplicationSet.

Required Secret

Create the secret out-of-band (not stored in Git):

kubectl create secret generic llm-pc-broker-secrets \
  --namespace llm-pc-broker \
  --from-literal=PC_MAC='aa:bb:cc:dd:ee:ff' \
  --from-literal=API_TOKEN='changeme' \
  --from-literal=SHUTDOWN_AGENT_TOKEN=''
